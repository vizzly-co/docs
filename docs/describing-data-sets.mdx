# Data sets

A data set in Vizzly, defines fields that describe the structure of data made available to build charts from. In a data set
definition you'll find attributes such as field names, field data types and the public name of the field your users will see when
using the dashboard.

For example, a simple data set might look like:

```json
{
  "id": "das_e400b5180d9f419db9418510383cfb0e",
  "name": "US Sales",
  "fields": [
    {
      "name": "id",
      "dataType": "number",
      "publicName": "User ID",
      "collection": "users"
    },
    {
      "name": "full_name",
      "dataType": "string",
      "publicName": "Full name",
      "collection": "users"
    },
    {
      "name": "born_on",
      "dataType": "date_time",
      "publicName": "Born on",
      "collection": "users"
    },
  ],
  "joins": []
}
```

## Data set field properties

| Property | Description                                         |
|----------|-----------------------------------------------------|
| id       | Unique identifier of the data set                   |
| name     | Name of the data set, seen when crafting dashboards |

#### Big query specific

| Property  | Description                                                                        |
|-----------|------------------------------------------------------------------------------------|
| namespace | Namespace of the data set, in the format `<<google project id>>.<<data set name>>` |


### SQL integration field properties

| Property         | Description                                     |
|------------------|-------------------------------------------------|
| field.name       | Field name                                      |
| field.dataType   | "number", "boolean", "string" or "date_time"    |
| field.publicName | Name of the field seen when crafting dashboards |
| field.collection | Table name                                      |
